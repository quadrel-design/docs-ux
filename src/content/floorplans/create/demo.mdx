# Floorplan - Create

## Create an initial Business Object

The create flow defines a standardized, step-based process for initially creating a new business object from a table-based overview page. It starts with a dedicated entry action on the overview (e.g. “New Business Object”), opens a multi-step wizard with a fixed, non-dynamic step sequence, and enforces functional validation on each step before progress or submission is allowed. A cancel dialog ensures that premature termination is always explicit and that the user is returned in a controlled way to the overview. After a successful submit, the new business object is persisted, the system navigates back to the table, and a non-blocking success snackbar confirms completion of the create process.

### Validation and Progress

Before moving to the next step or performing the final submit, all mandatory fields of the current step are validated. As long as required fields are missing or invalid, the navigation (Next/Submit) remains disabled; error states are shown directly on the affected fields and make the functional validation rules visible.
